**HabitX/HabitXWidget/Intents/HabitEntity.swift:**
- `import AppIntents`
- `struct HabitEntity: AppEntity` with fields: `var id: UUID`, `var name: String`, `var habitType: String`, `var unit: String`

- Per D-04: this entity is used for the small widget's AppIntentConfiguration picker dropdown

**HabitX/HabitXWidget/Intents/HabitEntityQuery.swift:**
- `import AppIntents`, `import SwiftData`
- `struct HabitEntityQuery: EntityQuery`
- `@MainActor func suggestedEntities() async throws -> [HabitEntity]` — fetches all habits from `SharedModelContainer.container.mainContext` using `FetchDescriptor<HabitSchemaV1.Habit>(sortBy: [SortDescriptor(\.sortOrder)])`, maps each to HabitEntity
- `@MainActor func entities(for identifiers: [UUID]) async throws -> [HabitEntity]` — fetches habits by UUID using `#Predicate { ids.contains($0.id) }`, maps to HabitEntity
- CRITICAL: Use `@MainActor` on both methods — `mainContext` is MainActor-isolated (Swift 6 strict concurrency)
- CRITICAL: Do NOT use @Query — it is unavailable in non-SwiftUI contexts

**HabitX/HabitXWidget/Intents/HabitWidgetIntent.swift:**
- `import AppIntents`
- `struct HabitWidgetIntent: WidgetConfigurationIntent`
- `static var title: LocalizedStringResource = "Select Habit"`
- `static var description = IntentDescription("Choose a habit to track.")`
- `@Parameter(title: "Habit") var habit: HabitEntity?`
- Per D-04: this intent powers the small widget's edit-mode picker

**HabitX/HabitXWidget/Intents/ToggleBooleanHabitIntent.swift:**
- `import AppIntents`, `import SwiftData`, `import WidgetKit`
- `struct ToggleBooleanHabitIntent: AppIntent`
- `static var title: LocalizedStringResource = "Toggle Habit"`
- `@Parameter(title: "Habit ID") var habitId: String` (UUID as string — AppIntent @Parameter requires String)
- `@MainActor func perform() async throws -> some IntentResult`:
  1. Create `let context = ModelContext(SharedModelContainer.container)`
  2. Fetch habit by UUID: `let descriptor = FetchDescriptor<HabitSchemaV1.Habit>(predicate: #Predicate { $0.id.uuidString == idStr })` where `let idStr = habitId`
  3. Guard unwrap first result, return `.result()` if not found
  4. Get `let today = Calendar.current.startOfDay(for: Date())`
  5. Check if log exists for today: `habit.logs.first(where: { $0.date >= today && $0.date < Calendar.current.date(byAdding: .day, value: 1, to: today)! })`
  6. If exists: `context.delete(existing)`. If not: create `HabitSchemaV1.HabitLog()` with `date = today`, `value = 1.0`, `habit = habit`, insert into context
  7. CRITICAL: Call `try? context.save()` — auto-save does NOT fire outside SwiftUI environment
  8. Call `WidgetCenter.shared.reloadAllTimelines()` per D-11
  9. Return `.result()`

**HabitX/HabitXWidget/Intents/IncrementCountHabitIntent.swift:**
- Same structure as ToggleBooleanHabitIntent
- `static var title: LocalizedStringResource = "Increment Habit"`
- In `perform()`: fetch habit same way, then:
  - If today log exists: `existing.value += 1.0; existing.loggedAt = Date()`
  - If not: create log with `value = 1.0`
  - Call `try? context.save()` then `WidgetCenter.shared.reloadAllTimelines()`

Update project.yml with two changes:

**1. Register habitx:// URL scheme for the HabitX app target (per D-12, WID-04):**

Under the `HabitX` target's `settings.base`, add:
```yaml
INFOPLIST_KEY_CFBundleURLTypes: |
  (
    {
      CFBundleURLName = "com.habitx.app";
      CFBundleURLSchemes = (habitx);
    }
  )
```

Note: With `GENERATE_INFOPLIST_FILE: true`, Xcode reads URL types from the INFOPLIST_KEY_ build setting. The plist value format is an array of dictionaries in old-style plist notation (parentheses for arrays, curly braces for dicts). This is the standard xcodegen pattern for URL schemes with generated Info.plist.
