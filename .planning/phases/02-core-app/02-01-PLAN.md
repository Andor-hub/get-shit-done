```swift
enum HabitSchemaV1: VersionedSchema {
    @Model final class Habit {
        var id: UUID = UUID()
        var name: String = ""
        var habitType: String = "boolean"  // "boolean" | "count" | "input"
        var dailyTarget: Double = 1.0
        var unit: String = ""
        var reminderTime: Date? = nil
        var createdAt: Date = Date()
        var sortOrder: Int = 0
        @Relationship(deleteRule: .cascade, inverse: \HabitLog.habit)
        var logs: [HabitLog] = []
        init() {}
    }

    @Model final class HabitLog {
        var id: UUID = UUID()
        var date: Date = Date()
        var value: Double = 0.0
        var loggedAt: Date = Date()
        var habit: Habit? = nil
        init() {}
    }
}

enum HabitType: String, CaseIterable, Sendable {
    case boolean
    case count
    case input
}
```

  <behavior>
    HabitLogService (operates on ModelContext + Habit):
    - toggleBoolean: If today's log exists for habit, delete it. If not, create HabitLog with date=startOfDay, value=1.0.
    - incrementCount: If today's log exists, add 1.0 to value. If not, create HabitLog with value=1.0.
    - setValue(habit, value, context): If today's log exists, replace value. If not, create HabitLog with that value.
    - todayValue(for habit) -> Double: Sum all log values where log.date is in today's date range. Returns 0 if no log.
    - isCompleted(habit) -> Bool: todayValue >= dailyTarget.
    - All date boundaries use Calendar.current.startOfDay(for: Date()) and +1 day for end-of-day. Never raw seconds.

    StatsCalculator (pure functions, no ModelContext):
    - currentStreak(for habit) -> Int: Walk backward from today. If today is completed, count from today. If today is NOT completed, count from yesterday. If yesterday is also not completed, return 0.
    - bestStreak(for habit) -> Int: Scan all logs sorted by date, find longest consecutive calendar-day run where value >= dailyTarget. Deduplicate days first (Set of startOfDay dates).
    - completionRate30Days(for habit) -> Double: Count distinct completed days in last 30 calendar days / 30.0.
    - completionByDay(for habit, days: Int) -> [(date: Date, completed: Bool)]: Returns array of last N days with completion status for chart data.

    HabitDefaults:
    - struct HabitTemplate with name, habitType, dailyTarget, unit fields.
    - static let proteinTemplate = HabitTemplate(name: "Protein", habitType: .input, dailyTarget: 150.0, unit: "g")
    - static let waterTemplate = HabitTemplate(name: "Water", habitType: .count, dailyTarget: 8.0, unit: "cups")
    - static let defaultHabits: [HabitTemplate] = [proteinTemplate, waterTemplate]
    - func createHabit(from template: HabitTemplate, sortOrder: Int) -> HabitSchemaV1.Habit (creates a new Habit model from template)
    - App accent color: define as a SwiftUI Color constant. Use Color(red: 0.0, green: 0.6, blue: 0.7) — a calm teal that works in both light and dark mode (per D-12, Claude's discretion on exact color).
  </behavior>
  <action>
    Create three new directories and files:

    1. `HabitX/HabitX/Services/HabitLogService.swift` — An enum with static functions (no instance state needed). Each function takes a `HabitSchemaV1.Habit` and `ModelContext`. Use the one-log-per-day upsert pattern from RESEARCH.md Pattern 4. For todayValue and isCompleted, these are read-only helpers that operate on `habit.logs` without needing ModelContext. All date math uses `Calendar.current.startOfDay(for: Date())`. The "today" boundary is always computed fresh (not cached) to handle midnight rollover (per LOG-04, Pitfall 4 from research).

    2. `HabitX/HabitX/Utilities/StatsCalculator.swift` — An enum with static pure functions. Takes `HabitSchemaV1.Habit` (reads `.logs` and `.dailyTarget`). For currentStreak: build a Set of completed startOfDay dates, then walk backward from today (if completed) or yesterday (if not). For bestStreak: sort all completed days, iterate tracking consecutive runs. For completionRate30Days: count completed days in last 30 / 30.0. For completionByDay: generate date array for last N days, check each against the completed set. Always convert HabitType via `HabitType(rawValue:)` — never switch on raw string (per Pitfall 6).

    3. `HabitX/HabitX/Utilities/HabitDefaults.swift` — struct HabitTemplate (Sendable) with the two premade habits. The createHabit function instantiates a HabitSchemaV1.Habit(), sets all fields from template, and sets sortOrder. Define `appAccentColor` as a static Color constant.

    IMPORTANT: These files must be added to the HabitX app target only (not widget). The widget extension in project.yml already includes HabitX/HabitX/Models — it does NOT include Services/ or Utilities/ and should not need them until Phase 3.

    Ensure all types are Sendable where needed for Swift 6 strict concurrency. HabitLogService functions that take ModelContext are @MainActor since ModelContext is not Sendable.
