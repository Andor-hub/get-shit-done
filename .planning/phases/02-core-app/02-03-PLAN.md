    - HabitHistoryView shows 90 days of history for one habit

    - Boolean: "Completed" or "Not completed"

  <action>
    Create directory: Features/History/.

    1. **HistoryListView.swift**: NavigationStack with title "History". @Query(sort: \HabitSchemaV1.Habit.sortOrder) var habits. If habits.isEmpty, show ContentUnavailableView("No Habits", systemImage: "clock", description: Text("Add habits from the Today tab to see history here.")). Otherwise, List of habits with NavigationLink to HabitHistoryView(habit:). Each row: HStack with habit name (.headline) and habit unit/type as subtitle (.subheadline, .secondary).

    2. **HabitHistoryView.swift**: Takes habit: HabitSchemaV1.Habit. Generate an array of the last 90 dates: use a computed property or onAppear that builds `[Date]` from today back 90 days using `Calendar.current.date(byAdding: .day, value: -i, to: Calendar.current.startOfDay(for: Date()))`. For each date, look up the log: `habit.logs.first(where: { Calendar.current.isDate($0.date, inSameDayAs: date) })`. Display in a List with sections or plain rows.

    Each row layout:
    - Leading: date formatted with `.dateTime.weekday(.abbreviated).month(.abbreviated).day()` (e.g. "Sat, Mar 29")
    - Trailing: value text based on habit type:
      - Boolean: if log exists with value >= 1.0, show checkmark + "Completed" in accent color; else "Not completed" in secondary color
      - Count: if log exists, show "\(Int(log.value))/\(Int(habit.dailyTarget)) \(habit.unit)"; else "No entry" in secondary
      - Input: if log exists, show "\(log.value, specifier: "%.0f")/\(habit.dailyTarget, specifier: "%.0f") \(habit.unit)"; else "No entry" in secondary
    - Completed days (value >= dailyTarget) get subtle accent tint on the row

    Navigation title: habit.name. Use .navigationBarTitleDisplayMode(.large).

    Performance note: 90 days x 1 lookup each is fine. habit.logs is already in memory from the relationship. Do NOT use a separate @Query for this — read directly from the habit's logs relationship (per research Pattern 3). Build a dictionary `[Date: HabitLog]` keyed by startOfDay for O(1) lookups instead of scanning the array 90 times.
  </action>
